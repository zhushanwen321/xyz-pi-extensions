# Rule Templates

## 核心格式：后果链

每条规则必须附带因果推理。缺后果链 = 裸规则，AI 会找绕过路径。

### 四种规则类型

**约束型：绝对不能做的行为**
```
禁止 {行为}：因为 {直接后果} → 导致 {连锁后果} → 最终 {最终影响}
```

**偏好型：优先选择的做法**
```
优先 {A} 而非 {B}：因为 {A 的优势} → 适用 {场景}。当 {例外条件} 时可用 {B}
```

**上下文型：特定场景的指导**
```
当 {条件} → 做 {行为} → 否则 {fallback}。因为 {不遵守的风险}
```

**反模式型：看似正确但有问题的做法**
```
不要用 {常见做法}（看起来合理，但实际效果是 {真实后果}）。改用 {正确做法}
```

## 写作规则

**密度优先**：删掉任何一句话，行为会变吗？不会 → 删掉。
"小心"、"确保"、"注意" — 不改变行为，必须删。

**边界条件**：每条规则必须有适用范围 + 例外。
"禁止 any" 没有边界，"禁止 any，第三方 API 类型声明除外" 有边界。

**规则数量**：CLAUDE.md 15-25 条为宜。少于 10 条 = 覆盖不足，
超过 40 条 = 密度崩塌，遵守率随数量上升而下降。

**载体感知**：不同载体用不同风格

| 载体 | 规则风格 | 侧重 |
|------|---------|------|
| CLAUDE.md | 约束型 + 偏好型 | 原则 + 禁令 |
| SKILL.md | 流程型（触发 → 退出条件 → 产出） | 触发条件 + 退出条件 |
| Hook 规则 | 事件 → 条件 → 动作 | 事件响应 |
| Agent prompt | 角色 + 能力边界 + 沟通风格 | 身份定义 |

## 反模式检测

写完规则后逐条检查：

| 反模式 | 检测信号 | 修复 |
|--------|---------|------|
| 空泛规则 | "确保"、"小心"、"注意" | 换成可验证的具体行为 |
| 裸规则 | "禁止 X"，无后果链 | 补充后果链 |
| 缺边界 | 只有条件无例外 | 加"当 {例外} 时可以" |
| 重复 | 同一意思出现两次 | 合并或删一条 |
| 冲突 | A 说"做 X"，B 说"不做 X" | 选一个，说明理由 |

## 质量检查

1. **后果链检查**：每条规则都有"为什么"？有裸规则 → 补充
2. **反模式检查**：逐条过清单，发现问题立即修复
3. **可验证性检查**：看代码时能判断规则是否被遵守？不能 → 重写
4. **边界检查**：每条规则都有适用范围 + 例外？没有 → 补充

## 反例对比

**差 — 裸规则：**
```
禁止使用 any，用 unknown 或具体类型
```
问题：只说了"做什么"，没说"为什么"。AI 不理解原因，会找绕过路径。

**好 — 后果链：**
```
禁止 any：any 禁用类型检查 → 编译器不再捕获类型错误 →
错误推迟到运行时 → 在 AI 生成的代码中尤其危险（AI 倾向
用 any 绕过它不理解的部分）。用 unknown 或具体类型替代。
与第三方 API 交互时可适当放宽。
```
好在哪：后果链 + 替代方案 + 例外条件。
