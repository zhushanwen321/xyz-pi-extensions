## Budget 加权计算抽取 shared 模块（中等方案）

### 目标

把两个扩展共享的 token 加权计算口径抽取到 shared 包，同时修复 goal 的 `accumulateTokens` bug（`max(input - cacheRead, 0)` 基于「input 包含 cacheRead」的错误假设，pi 的 input/cacheRead 互斥）。

### 新建 shared 包：`shared/budget-accounting/`

包名：`@zhushanwen/pi-budget-accounting`（published，因为 goal/workflow 都是 published 包要在 dependencies 声明）。

**文件清单**（参照 quota-providers 模式）：
- `package.json` — name/type/main/files/keywords/scripts，无 peerDeps（零 Pi 依赖纯计算）
- `index.ts` — re-export `src/index.js`
- `src/index.ts` — barrel export
- `src/accounting.ts` — 权重常量 + weightTokens + TokenConsumption 类型
- `src/__tests__/accounting.test.ts` — weightTokens 单元测试
- `tsconfig.json` — extends 根 tsconfig
- `vitest.config.ts` — 标准 include

**`src/accounting.ts` 内容**：
```ts
/** Token 消耗输入（最小公共类型）。
 * goal 的 TokenUsage 和 workflow 的 AgentUsage 都结构兼容此类型。 */
export interface TokenConsumption {
  input: number;
  output: number;
  cacheRead: number;
  cacheWrite: number;
}

/** Budget 加权系数。详见 budget.ts 上方说明。 */
export const INPUT_WEIGHT = 1;
export const CACHE_READ_WEIGHT = 0.02;
export const CACHE_WRITE_WEIGHT = 0;
export const OUTPUT_WEIGHT = 2;

/** 四项 token 加权求和。input/cacheRead 互斥（pi provider 保证），无需去重。 */
export function weightTokens(c: TokenConsumption): number {
  return c.input * INPUT_WEIGHT
       + c.output * OUTPUT_WEIGHT
       + c.cacheRead * CACHE_READ_WEIGHT
       + c.cacheWrite * CACHE_WRITE_WEIGHT;
}
```

### goal 改动（2 文件 + 2 测试）

**`extensions/goal/src/engine/budget.ts`**（核心 bug 修复）：
- accumulateTokens 从 `@zhushanwen/pi-budget-accounting` import weightTokens
- Before（有 bug）：
  ```ts
  if (input > 0 || output > 0) {
    return currentTokensUsed + Math.max(input - cacheRead, 0) + output;
  }
  return currentTokensUsed + (usage.totalTokens ?? 0);
  ```
- After（修复 + 加权）：
  ```ts
  if (input > 0 || output > 0) {
    return currentTokensUsed + weightTokens({ input, output, cacheRead, cacheWrite: 0 });
  }
  // fallback：totalTokens 无法区分四桶，按 input 权重 1（基准）保守估算
  return currentTokensUsed + (usage.totalTokens ?? 0) * INPUT_WEIGHT;
  ```
  注意：cacheWrite 不在 goal 的 TokenUsage 接口里，硬传 0（goal 不追踪 cacheWrite）。
  fallback 分支保留（防御性，按 INPUT_WEIGHT=1 计算）。
