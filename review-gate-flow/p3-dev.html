Phase-Gate（2 步，阶段二通过后自动触发）

步骤执行者说明
1. 脚本检查脚本文档完整性 + YAML frontmatter + placeholder 扫描（最多 5 次）
2. 防伪造检查AI Agent（gate-reviewer subagent）验证 review 报告非空、test_results 包含实际命令输出、代码变更真实存在
FAIL → 打回主 agent 修复 → 修复后直接重新提交 phase-gate（跳过 review-gate）
Phase 3 的防伪造检查比 Phase 1/2 更严格：额外验证代码变更真实存在、测试结果包含实际命令输出。
