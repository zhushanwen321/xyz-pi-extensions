Playwright 并行说明

维度策略
browser 进程每个 frontend 测试 subagent 启动独立 Playwright browser
数据隔离每个 subagent 使用独立测试用户账号
调试手段subagent 可通过 browser-automation skill 截图定位问题
并发限制同一 Wave 内最多 2 个 Playwright subagent（避免资源争抢）

前端测试 Subagent 的 Skill 注入（强制）

强制加载的 Skill / MCP用途
browser-automation截图、元素检查、UI 交互、样式检查
playwright-mcpPlaywright 测试执行和断言
chrome-devtools网络监控、性能分析、Console 日志
主 agent 在 dispatch 前端测试 subagent 时，task prompt 中必须包含“加载上述 3 个 skill”的指导。Subagent 进程隔离，每个 subagent 的 Playwright browser 完全独立，不能共享页面状态。

主 Agent 与 Subagent 的职责分离

角色职责
主 Agent启动 dev server / backend server / 数据库等基础设施。确保所有服务就绪后再 dispatch subagent
Subagent（API 测试）直接通过 curl/httpx 调用已启动的服务，验证接口行为
Subagent（前端测试）连接主 agent 启动的 dev server，启动独立 Playwright browser 进程执行页面操作和断言
Subagent（集成测试）端到端调用多个服务，验证模块协作
关键约束：主 agent 启动服务，subagent 消费服务。Subagent 不负责启动/停止服务器。前端测试的每个 subagent 必须在不同进程中运行（Pi subagent 天然进程隔离）。
