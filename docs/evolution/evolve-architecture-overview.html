数据流：从原始数据到优化建议
1
Session 产生数据 — Pi 每次会话产生 JSONL 文件，包含所有消息、工具调用、compact 事件。存储在 ~/.pi/agent/sessions/*.jsonl
2
L2 实时追踪（session 内）— TypeScript Detectors 监听 session_compact 和 tool_result 事件，匹配后通过 pi.appendEntry() 写入 session entries。
3
L3 离线分析（每日首次 session）— pi.on("session_start") 触发 analyze.py，读取最近 1 天的 JSONL 文件，运行 6 个 Extractor + 14 条 Rule，产出 daily-reports/YYYY-MM-DD.json。
4
Extractor 自动发现 — Python 用 pkgutil.iter_modules 扫描 extractors/ 和 rules/ 目录，新增追踪维度只需添加 1 个 Python 文件，无需修改框架代码。
5
Rule 规则引擎 — 每个 Rule 文件实现 check(daily_report) → issues[]，接收所有 Extractor 的合并结果，独立判断是否触发问题。
6
L4 AI 分析 — /evolve Skill 读取 daily-reports/ JSON + 问题列表，通过 LLM 分析生成优化建议（如"优化 CLAUDE.md 中的工具描述"），写入 history.jsonl 供后续回顾。
2 个消费 Skill

/evolve — 进化分析

触发词: /evolve, 进化分析, analyze usage

读取每日分析报告，LLM 按优先级分析 6 个维度数据，生成可执行的优化建议。建议需要人工确认后才生效（human-in-the-loop）。支持建议的 apply / skip / rollback 生命周期管理。

/evolve-report — 统计报告

触发词: /evolve-report, 查看报告, 进化报告

展示每日分析报告的统计摘要：session 数量、工具使用分布、token 消耗趋势、追踪维度的关键指标。用于快速了解 Agent 使用模式，不触发 LLM 分析。
