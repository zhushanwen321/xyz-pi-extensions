# ADR-008: 段边界仅依据 user message

## 上下文

树压缩的基本单元是 Segment——一组相关的 turn 的集合。段边界的判定方式决定压缩粒度和准确性。两个选择：

- **Option A**: 语义分割：LLM 分析对话内容，根据"用户意图切换"动态划定段边界
- **Option B**: 机械分割：每条新的 user message 即为段边界

## 决策

选择 Option B：Segment 边界仅由新的 user message 触发。

## 原因

1. **可靠性优先于精度**：语义分割依赖 LLM 判断"意图是否改变"，但 LLM 的判断本身可能出错（把调试追问误判为新任务，或把独立子任务误判为继续）。语义分割引入了一个"需要 LLM 判断什么是 LLM 的意图"的循环依赖
2. **确定性**：user message 是明确的、可观测的信号。调试行为和监控行为完全可预测
3. **实现成本**：`turn_end` 事件中检测 messages 数组的 user role 数量变化即可，不需要额外 LLM 调用或复杂启发式
4. **可补救**：LLM 在压缩时可以通过 group 节点将"应该在一起的多个段"合并，段边界过细的代价是可恢复的。语义分割过粗（把两个独立任务合并为一个段）则不可恢复
