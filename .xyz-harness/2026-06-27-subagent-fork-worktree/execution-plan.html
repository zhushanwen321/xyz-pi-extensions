TL;DR · 核心结论
  • 6 个 Wave 垂直切片：Wave 1 编译基石（types+状态机）→ Wave 2 五叶子模块并行 → Wave 3 编排分流并行 → Wave 4 汇合点串行 → Wave 5 外壳/文档并行 → Wave 6 CLI E2E + 验收 Gate。
  • 依赖从⑤时序图推导：功能 B 调用功能 A 的方法 → Wave(B) blocked_by Wave(A)。汇合点 SubagentService 必须 last 串行（唯一改 subagent-service.ts）。
  • 测试双层落地：unit/integration 走既有 vitest src/__tests__/；E2E 走 pi CLI 模式 ~/Code/test-pi-extension/ 沙盒（7 scenario 覆盖 UC-1~UC-7）。
  • Definition of Done = 测试验收清单 36 用例全绿。末尾验收 Wave（Wave 6）blocked_by 全功能 Wave，不绿 = 未完成。
  • Round 1 追踪 CONVERGED：6 gap 全修（2 笔误/文件影响 + 2 标注缺口 + 1 用例断链 + 1 层级漂移），manifest 36/36 = ⑤test-matrix 全量。
