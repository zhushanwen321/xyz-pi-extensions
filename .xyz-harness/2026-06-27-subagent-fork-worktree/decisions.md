## 待 architecture 阶段解决的开放问题（非决策，记录为约束输入）

以下来自 design-clarity 审查 subagent 挖出的真盲区，已在 clarity 决策中给出方向，具体架构（模块划分/状态机/接口）留给 ②architecture：

- **worktree 清理三路径覆盖**（kill -9/配置失败/cancel 窗口期）+ 启动期 reaper（孤儿 worktree 清扫）→ 需新增 Worktree 生命周期管理器模块 + 进程启动期扫描
- **clean working tree 前置校验**（git status --porcelain，脏则 stash/报错/降级）
- **worktree 内 node_modules/.env 缺失**（symlink node_modules + setupHook 机制，参考 worktree.ts:177-188,270-313）
- **fork 与 compaction 写入交错**（fork 读源文件时主 agent 正在 compaction → 原子快照或只取 compaction 后路径）
- **嵌套委派语义**（worktree-of-worktree，首版可禁止 worktree 嵌套；fork 可嵌套受 D-007 深度限制）
