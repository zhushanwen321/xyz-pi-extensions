## 技术决策和取舍

1. **RPC 模式透传 vs 进程内代理**：选择 RPC 模式透传，subagent 通过 JSON-RPC 与主 agent 通信，保持进程隔离。代价是增加了消息序列化开销，但换来了清晰的进程边界。

2. **UI 请求队列**：多个 subagent 可能同时发起 ask_user 请求，采用队列机制保证请求串行化处理，避免 UI 冲突。队列深度不做硬限制（实际场景并发 ask_user 频率很低）。

3. **系统提示词注入时机**：在 subagent session 初始化时注入 ask_user 工具描述，确保 LLM 知道该工具可用。注入内容精简，避免膨胀 context。

## 遇到的问题和解决方案

1. **Review 发现 3 个 must-fix**：R1-R3 涉及错误处理边界和类型安全，修复后通过全量测试验证。问题主要集中在 RPC 断连时的状态清理和类型守卫缺失。

2. **集成测试环境依赖**：TC-E1 需要真实 Pi 环境，无法在纯 mock 环境运行。通过标记 `requiresRealEnv` 隔离，CI 中单独配置。
