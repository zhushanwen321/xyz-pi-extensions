为什么这么分

mock 层不需要复杂环境，机器门直接跑最可靠（AI 编不了 coverage 产物）。real 层要起服务/真实 DB，一个 subagent 的多轮 session 才搞得定，但它产出的报告可伪造——所以机器门只做「报告完整性校验 + 用户确认凭证」。

关键事实

worker 崩溃重试会 rebuildRuntime（源码 error-recovery.ts）——整个线程重建，内存变量丢失。所以解析结果必须落盘，不能只靠内存。run.meta 的重试计数跨重建存活（框架保证）。

为什么 git 必须串行

三个 implementer 同时 lightmerge add 会抢同一个 git index（.git/index 是单点资源），产生锁冲突或索引损坏。并行写码靠文件系统隔离（各自 worktree），串行合并靠 JS for 循环序列化——这是脚本控制流的事，不需要特殊机制。

五、开发循环内部（Q3 失败谁修）

implementer 写完某一层 → 跑覆盖率 → 没过就派 implementer 补，同一层内循环。
