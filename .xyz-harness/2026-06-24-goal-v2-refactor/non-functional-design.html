关键副作用（需下游 code-architecture 关注）

#5 budget 检查点位置修正

代码取证确认：事件路径走 persistAndUpdate（tickState + appendEntry + updateWidget），不走 service.persistState。budget 终态检查必须挂在事件路径的 persist 函数内（persistAndUpdate 或 #4 拆分后的等价函数），不是 persistState。

影响：#5 验收标准需修正——检查点在 persistAndUpdate，非 persistState。#4 拆分时 budget 检查逻辑随之迁移。

#9 __goalInit tasks 废弃影响三方

代码取证：__goalInit 有 2 个调用方——coding-workflow（主力，Phase 2/3 传真实 taskList）+ plan。均用 inline alias（非 import type），签名变更编译期不报错 → 静默 drift。

缓解：goal 侧容忍（忽略 tasks 参数不 throw）+ 同步迁移两个调用方 + 推动改用 import type。GoalInitBudget.maxTurns 需同步清理（#6 连带）。

#6 删自动终态后的兜底依赖

agent 不调 complete 时，唯一兜底是 budget 耗尽（persistAndUpdate 内检查）。这是 D21 决策的有意代价（对齐 Codex）。残余风险可接受：budget 兜底 + followUp 提示 + 手动 clear。
