## 章节结构

```markdown
# {需求标题}

- G1: {目标} — 成功标准: {可衡量指标}
  - G1.1: {子目标}
- G2: {目标}

### 达成路线
| 目标 | 路线/策略 | 对应用例 |
|------|---------|---------|
| G1 | {如何达成} | UC-1, UC-2 |

## 2. 业务用例（Use Cases）

### 用例图
（Mermaid use case diagram — Actor + 用例 + 系统边界）

### UC-1: {用例名称}
- **Actor**: {谁}
- **前置条件**: {什么情况下}
- **主流程**: 1. ... 2. ... 3. → 完成
- **替代流程**: {分支}
- **异常流程**: {错误处理}
- **后置状态**: {完成后系统状态}
- **关联目标**: G1

## 3. 数据流转（Data Flow）

### 数据流图
（Mermaid 或文字 DFD — 数据源 → 处理 → 存储 → 消费）

### 数据清单
| 数据 | 来源 | 处理 | 消费者 | 归档策略 | 敏感级别 |
|------|------|------|--------|---------|---------|

## 4. 功能清单（Features）

## 5. UI/UX 场景（Interface Scenarios）

> 纯后端/API 系统标注「无 UI 交互」，跳过本节。

### 页面线框（文字描述）
（关键页面的布局/交互描述）

|---------|---------|---------|-----------|
```
