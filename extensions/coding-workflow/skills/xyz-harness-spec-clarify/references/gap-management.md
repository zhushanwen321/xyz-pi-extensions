# Gap 管理：发现、分类、解决

Gap 是收敛循环的燃料。追踪卡住的地方就是遗漏。每个 gap 都是避免后续返工的机会——宁可多发现 10 个不必要的，也不要漏掉 1 个关键的。

## Gap 分类

| 类型 | 含义 | 来源 | 解决方式 |
|------|------|------|---------|
| **F** (Fact) | 代码中有但我还没读到的信息 | 模型中的 `[UNVERIFIED]` 标记 | dispatch on-demand scan |
| **K** (Knowledge) | 只有用户知道的业务规则 | 追踪卡住且代码无答案 | 生成具体问题问用户 |
| **D** (Decision) | 需要做选择的权衡点 | 多种方案都可以，不确定选哪个 | 生成方案对比问用户 |

## Gap 优先级

| 优先级 | 条件 | 示例 |
|--------|------|------|
| **P0** | 影响核心流程的完整性 | "创建订单时，库存不足怎么办？" |
| **P1** | 影响边界/异常场景 | "并发修改同一订单的处理策略" |
| **P2** | 影响优化/扩展场景 | "未来需要支持批量操作吗？" |

P2 gap 可以 defer，不阻塞收敛。

## Gap Tracker

嵌入 `clarification.md`，跟踪每个 gap 的状态流转。

```markdown
## Gap Tracker

| ID | Type | Priority | Source | Question | Status | Resolution |
|----|------|----------|--------|----------|--------|------------|
| G-001 | K | P0 | P1/OP-U01 | 提交订单需要二次确认吗？ | open | — |
| G-002 | D | P1 | P3/OP-A01 | 重复提交用 debounce 还是 idempotency key？ | resolved | idempotency key (order_hash) |
| G-003 | F | P0 | P2/E01 | Order.status 的完整枚举值？ | open | — |
```

## Gap 解决（Step 7）

按优先级从高到低解决：

**F 类型（事实）：** dispatch on-demand subagent 扫描代码，获取答案，更新模型。标记 `[VERIFIED: {来源}]`。

**K 类型（知识）：** 生成具体的、有上下文的问题问用户。
- 不要问 "退款怎么处理？"
- 要问 "我看到代码里有 `refund` 表但状态只有 `pending` 和 `completed`——退款是部分退款还是全额退款？退款后订单状态变成什么？"

**D 类型（决策）：** 生成方案对比（2-3 选项 + trade-off），让用户选择。记录决策 + 推理过程。

**用户回答后：**
1. 更新模型对应维度
2. 更新 Gap Tracker（status → resolved）
3. 检查是否有新 gap 被触发（用户回答可能引入新问题）
4. 如果有新 gap → 记录并继续解决

## 追踪时的"卡住信号"

以下情况说明你遇到了 gap：

1. **"我不知道"** → 你需要信息才能继续追踪
2. **"大概是..."** → 你在猜测，不是在追踪
3. **"应该可以..."** → 你在假设，不是在确认
4. **"这不重要"** → 可能重要，标记为 P2
5. **"这取决于..."** → 有未做的决策
6. **"让我看看代码"** → F 类型 gap，需要扫描

## Gap 密度指标

| 阶段 | 预期 Gap 密度 | 说明 |
|------|--------------|------|
| Round 2（首次追踪） | 高（10-30 个） | 正常——模型刚建立 |
| Round 3 | 中（3-10 个） | 核心问题已在 Round 2 暴露 |
| Round 4 | 低（0-3 个） | 接近收敛 |
| Round 5+ | 极低（0 个） | 应该已收敛 |

如果 Round 4+ 仍有 >3 个新 gap → 可能是 scope 太大，考虑 decompose 为子系统。

## Stagnation 判定

连续 3 轮 gap 数量不降（新发现 ≥ 已解决）的可能原因：

1. **Scope 过大** → 需要 decompose 为子系统
2. **用户回答引入新问题** → 正常但需要控制深度
3. **AI 在反复追问相同领域** → 聚焦不够，需要缩小追踪范围

处理：强制收敛，将未解决的 gap 标记为 `[UNRESOLVED]`，写入 spec。详见 `convergence-loop.md` 的收敛条件章节。
