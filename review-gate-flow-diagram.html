Review-Gate 机制详解

触发方式

主 agent 完成当前阶段产出后，调用 coding-workflow-review-gate(phase=N)。
不再由主 agent 自行发起 review。

内部状态

维护 {topic_dir}/.review-gate.json，记录每轮的发现/修复数量。
JSON 结构：{round, total_issues, must_fix, fixed, reviewer_md}

循环逻辑

① dispatch reviewer subagent → ② 解析 must_fix → ③ >0 则注入修复指导给主 agent → ④ 主 agent 修复后重新调用 review-gate → ⑤ must_fix=0 时通过

终止条件

• must_fix = 0 → 通过
• 最大 3 轮 → 强制通过（附带警告）
• 连续 2 轮 must_fix 不降 → 人工介入

通过后流转

• 如果后面还有 step → 反馈主 agent 执行下一步
• 如果是最后一步 → 自动提交到 phase-gate（coding-workflow-gate）

Subagent 构造

通过 runSingleAgent() + system prompt（读取对应 reviewer skill 的 SKILL.md）构造。每个 phase 有不同的 reviewer skill 组合。

{ "phase": 3, "rounds": [ { "round": 1, "reviewer": "xyz-harness-business-logic-reviewer", "must_fix": 3, "fixed": 3 }, { "round": 1, "reviewer": "xyz-harness-standards-reviewer", "must_fix": 1, "fixed": 1 }, { "round": 1, "reviewer": "xyz-harness-robustness-reviewer", "must_fix": 0, "fixed": 0 }, { "round": 2, "reviewer": "xyz-harness-business-logic-reviewer", "must_fix": 0, "fixed": 0 } ], "status": "passed", "total_rounds": 2 }

可行性分析

1. runSingleAgent() 是否支持？ ✓ 完全支持

现有 review-dispatcher.ts 已用 runSingleAgent() dispatch gate 反欺诈 reviewer。review-gate 只需复用同一机制，改变 system prompt（从 gate-reviewer 换成 content-reviewer）。

2. 每个 phase 的 reviewer 如何区分？ ✓ 通过 system prompt + task prompt 区分

runSingleAgent() 接受 systemPrompt 和 task 参数。不同 phase 传入不同的 reviewer skill 路径（如 Phase 1 用 expert-reviewer 的计划评审模式，Phase 3 用 5 个专项 reviewer），无需为每个 phase 创建独立的 agent.md。

3. 循环控制如何实现？ ✓ 可行，需新增 tool

新增 coding-workflow-review-gate tool，内部维护 JSON 状态文件。该 tool 替代各 SKILL.md 中的 review 步骤。主 agent 只需在合适位置调用该 tool，循环由 tool 内部自动管理。

4. Phase 3 的 5 维度并行审查如何实现？ ⚠ 需要改造 runSingleAgent 为 runParallelAgents

当前 subagent.ts 只支持 runSingleAgent()（单 subagent）。Phase 3 的 Batch 1（BLR+Standards+Taste+Robustness）需要并行 dispatch 4 个 subagent。需要新增 runParallelAgents() 或在 review-gate tool 内部串行 dispatch（牺牲性能换简单性）。

5. SKILL.md 改造范围 ✓ 改动明确

• Phase 1/2/3/4 的 SKILL.md：删除 review 步骤（Spec Review / Plan Review / Five-Step Review / Test Review 章节），改为"完成后调用 coding-workflow-review-gate(phase=N)"
• Phase 5：不变
• review-dispatcher.ts：新增 dispatchContentReview() 方法
• tool-handlers.ts：新增 executeReviewGateTool() handler

6. 与现有 phase-gate 的关系 ✓ 职责分离清晰

• Review-Gate（新）：内容质量审查，循环 review-fix 直到无问题。审查者是 content reviewer subagent。
• Phase-Gate（现有）：脚本检查（tsc/eslint/gate.py）+ 反欺诈审查。审查者是 gate-reviewer subagent。
• 执行顺序：Review-Gate 通过 → 自动进入 Phase-Gate。
